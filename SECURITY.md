# Security policy

## Supported versions

Security fixes are applied to the latest published QuotaHalo release.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability or privacy problem.

Use GitHub's **Security → Report a vulnerability** flow for the `sanoobis/quotahalo` repository. Include affected versions, reproduction steps, impact, and any suggested mitigation. Reports will be acknowledged as soon as practical.

QuotaHalo reads local Codex session files. Reports involving unintended exposure of prompt content, credentials, authentication data, arbitrary file access, renderer privilege escalation, or unsafe update/distribution behavior are especially important.

# Contributing to QuotaHalo

Thank you for helping improve QuotaHalo.

## Before opening a pull request

1. Discuss substantial features in an issue first.
2. Keep changes focused and avoid unrelated formatting rewrites.
3. Run `npm test` and confirm the app starts with `npm start`.
4. Do not include session logs, prompts, authentication files, credentials, or other personal data.
5. Update documentation and tests when behavior changes.

## Development setup

```powershell
npm install
npm test
npm start
```

## Pull requests

Explain what changed, why it changed, the user impact, and how it was tested. Screenshots are encouraged for visible UI changes.

By intentionally submitting a contribution for inclusion in QuotaHalo, you agree to license that contribution under GPL-3.0-only and certify that you have the right to submit it.

Contributors are encouraged to sign commits using the Developer Certificate of Origin convention:

```text
Signed-off-by: Your Name <your-email@example.com>
```

Use `git commit -s` to add the sign-off automatically.
